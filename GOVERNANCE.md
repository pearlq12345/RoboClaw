# Governance

RoboClaw is an open, co-created project. We want important project directions to be discussed in public and, when appropriate, shaped through community voting.

## Decision Principles

- Keep major project decisions visible and documented
- Prefer GitHub-native workflows over custom infrastructure
- Separate open discussion from final implementation work
- Use community input to guide roadmap priorities
- Let maintainers keep the project coherent, safe, and shippable

## Where Decisions Happen

- `GitHub Discussions`: brainstorming, debate, Q&A, and polls
- `GitHub Issue Forms`: structured proposals that can move onto the roadmap
- `GitHub Projects`: accepted work and tracked priorities
- Pull requests: implementation of accepted decisions

## What The Community Can Influence

Community discussion and voting are especially encouraged for:

- Project logo, icon, and visual identity
- Which real robots to support first
- Which simulators to support first
- Public roadmap priorities
- Major developer-experience improvements
- Documentation and onboarding improvements

## What Maintainers Decide

Maintainers keep final responsibility for:

- Repository safety and moderation
- Security-sensitive decisions
- Release timing and branch management
- Technical decisions that require architectural consistency
- Accepting or declining proposals after community discussion

Maintainers should still explain important decisions in public whenever practical.

## Proposal Lifecycle

1. Start with a GitHub Discussion if the idea is still rough.
2. Submit a structured proposal through the relevant Issue Form once the idea is clear.
3. Maintainers may open a Discussion poll for proposals that need community prioritization.
4. Accepted proposals move into the roadmap or issue backlog.
5. Implementation happens through issues, pull requests, and review.

## Initial Community Priorities

The first public decision topics for RoboClaw are:

- The project logo and icon
- The first real robots to support
- The first simulators to support
- The first milestone feature set

## GitHub Setup Checklist

To fully enable this workflow in the repository settings:

1. Turn on `GitHub Discussions`
2. Create Discussion categories such as `Ideas`, `Polls`, `Q&A`, and `Announcements`
3. Create a public GitHub Project for roadmap tracking
4. Pin the first logo or icon poll after Discussions are enabled

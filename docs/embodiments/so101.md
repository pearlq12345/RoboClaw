# SO101 Embodiment Notes

## External SDK

RoboClaw no longer vendors `scservo_sdk` inside this repository.

For SO101 real-hardware workflows, install the SCServo Python SDK into the same
Python environment that launches:

- `roboclaw`
- the ROS2 control-surface server
- the SO101 serial probe module

RoboClaw expects the import name to remain:

```python
import scservo_sdk
```

If you already have the SDK source tree elsewhere, install it into the target
environment before running RoboClaw:

```bash
python -m pip install /path/to/scservo-sdk
```

If you keep a local editable checkout for development, that also works:

```bash
python -m pip install -e /path/to/scservo-sdk
```

## Validation

After installing the SDK, verify that both commands work in the target
environment:

```bash
python -c "import scservo_sdk; print('ok')"
python -m roboclaw.embodied.execution.integration.control_surfaces.ros2.scservo_probe /dev/serial/by-id/<your-device>
```

If the import fails, SO101 onboarding, probing, and ROS2 control-surface startup
will stop with a message pointing back to this document.

## Runtime Expectations

Install the SDK into every Python environment that will do any of the following:

- run `roboclaw`
- run the ROS2 control-surface server
- execute the SO101 serial probe on a target host or inside Docker

If RoboClaw reports that the SDK is missing, fix the environment first instead
of copying `scservo_sdk` back into this repository.

# Contributing

Thank you for helping build RoboClaw in the open.

## Start Here

- Use [GitHub Discussions](https://github.com/Elvin-yk/RoboClaw/discussions) for early ideas, questions, and community debate
- Use the [Issue Forms](https://github.com/Elvin-yk/RoboClaw/issues/new/choose) for structured proposals
- Use pull requests for implementation once a direction is accepted

## Proposal Types

We currently provide structured proposal forms for:

- Logo and icon ideas
- Real robot support requests
- Simulator support requests
- Feature proposals

## Good Proposal Guidelines

- Explain the problem clearly
- Explain why the proposal matters now
- Keep the first version small and practical
- Mention tradeoffs, risks, or open questions
- Link prior discussions when relevant

## How We Work

1. Discuss the idea openly
2. Turn it into a structured proposal
3. Gather community feedback or run a poll if needed
4. Move accepted work into the roadmap
5. Implement through pull requests

## Code Contributions

Before opening a larger pull request:

- Link the relevant discussion or proposal issue
- Keep scope focused
- Document user-facing changes
- Add tests when the change affects behavior

Please also read [GOVERNANCE.md](GOVERNANCE.md) and [SUPPORT.md](SUPPORT.md).
